# A股《可持续发展报告指引》浏览器（丸爸）

一个面向 A 股可持续发展报告编制与研究场景的指引浏览器。项目基于 Excel 拆解数据构建，采用纯静态前端实现，支持从"章-节-条-款-指引项"多层级快速定位内容。

**核心优势**：纯静态零后端 | 部署简单 | 打开即用 | 离线可用 | Excel 驱动

## 01 
## 主要功能  

### ①多层级导航浏览
- 左侧导航按章/节组织
- 右侧先显示「条」，点击条后展示对应「款」和「指引内容」
- 支持「附则/释义」等特殊结构直接浏览

### ②全文检索与定位
- 支持搜索章、节、条号、款号、指引内容和术语
- 搜索结果显示完整路径信息并支持点击跳转
- 跳转后自动定位到目标款并短暂高亮

### ③术语高亮与释义
- 正文中的专业术语自动蓝色高亮
- 鼠标悬浮即可查看术语定义，无需手动翻阅附则

### ④指引内容类型标签
每条指引内容均标注所属类型，支持鼠标悬浮查看说明：

| 标签 | 定义 |
|---|---|
| 原则 | 阐释了上市公司开展可持续发展相关工作的原则 |
| 指引应用规则 | 说明了上市公司应用指引的若干规则 |
| 披露操作要求 | 提出了开展可持续信息披露工作的基础性要求 |
| 披露框架 | 制定了披露议题内容的一般性框架 |
| 披露要点 | 明确了披露具体议题内容的要点 |
| 披露说明 | 对披露要点提供进一步的说明 |
| 释义 | 对指引涉及的术语进行定义 |

### ⑤双色披露责任标识
- **强制披露**：红色标签，明确显示法规义务要求
- **鼓励披露**：绿色标签，清晰区分自愿披露项目

### ⑥议题标签与快速定位
- 每条条文后显示所属议题（如「应对气候变化」「员工」「数据安全与客户隐私保护」等）
- 覆盖 21 项关键 ESG 议题，与指引中的完整披露覆盖面对齐

## 02
## 性能特点

- **零后端**：纯前端应用，无服务器依赖
- **快速加载**：构建期优化，术语预渲染
- **离线可用**：下载后完全离线使用
- **高响应性**：浏览器仅运行事件处理和 DOM 操作，无复杂计算

## 03
## 技术栈

- **前端**：原生 JavaScript（无框架依赖）+ HTML/CSS 3
- **数据处理**：Python 3 + openpyxl
- **构建**：Python 脚本
- **部署**：任何静态文件服务器

## 05
## 快速开始

### 方式一：使用预构建版本（推荐新手）

如果项目中已包含 `static/guideline-data.js`，直接启动即可：

```bash
# 1. 克隆项目
git clone https://github.com/wanziba/a_share_esg_guideline_brower.git
cd a_share_esg_guideline_brower

# 2. 启动本地服务器（推荐）
python -m http.server 8000

# 3. 在浏览器中打开
http://localhost:8000
```

**注意**：直接用浏览器打开 `index.html` 可能因 CORS 限制而加载失败，建议通过 HTTP 服务器访问。

### 方式二：从 Excel 源数据重新构建

如果需要从 Excel 数据重新生成 JavaScript 数据文件：

```bash
# 1. 克隆项目
git clone https://github.com/wanziba/a_share_esg_guideline_brower.git
cd a_share_esg_guideline_brower

# 2. 安装 Python 依赖（Python 3.7+）
pip install -r requirements.txt

# 3. 确保项目根目录下有 Excel 文件
# 文件名：沪深北交易所《可持续发展报告指引（试行）》要求拆解.xlsx

# 4. 运行构建脚本
python tools/build_guideline_data.py

# 5. 启动服务器
python -m http.server 8000

# 6. 在浏览器中打开
http://localhost:8000
```

构建脚本会自动：
- 解析 Excel 中的章、节、条、款、指引项等结构数据
- 进行术语高亮预渲染（在构建阶段而非浏览器运行时）
- 生成 `static/guideline-data.js` 文件
- 提取释义项并构建术语查询表


## 06
## 数据更新流程

当发布新版指引时，只需维护 Excel，无需改动任何前端代码：

1. **更新 Excel**：在项目根目录下编辑 Excel 文件，添加或修改条文内容
2. **重新构建**：运行 `python tools/build_guideline_data.py`
3. **刷新浏览器**：打开或刷新页面，新增条文、术语、类型即自动生效
